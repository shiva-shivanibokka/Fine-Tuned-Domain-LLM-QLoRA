# Frontend — Legal Clause Extraction Demo

Next.js 16 (App Router) UI for the fine-tuned Llama 3.2 3B CUAD model. Deployed on Vercel.

## Four views

- **Model Comparison** — paste a contract, extract a clause, see base vs. fine-tuned side by side. Calls the inference backend via a server route handler (`app/api/compare`).
- **Ablation Dashboard** — BERTScore / G-Eval / clause-accuracy / hallucination / ECE across LoRA → QLoRA → DPO, from `data/metrics.json`.
- **Failure Explorer** — UMAP + HDBSCAN failure clusters from `data/failures.json`.
- **Dataset** — clause distribution and split stats from `data/dataset_card.json` (real, generated by the data pipeline).

The three data-driven tabs read committed JSON snapshots, so only Model Comparison needs the live backend — the rest are static and instant.

## Local development

```bash
cd frontend
npm install
cp .env.local.example .env.local   # set HF_SPACE_URL to your Space
npm run dev                        # http://localhost:3000
```

## Deploy to Vercel

1. Import the repo in Vercel, set **Root Directory** to `frontend`.
2. Add environment variable **`HF_SPACE_URL`** = your Hugging Face Space URL.
3. Deploy. The static tabs work immediately; Model Comparison works once the Space is live.

## Updating the metrics/failure data

After a local training + evaluation run, refresh the committed snapshots:

```bash
cp ../data/processed/dataset_card.json data/dataset_card.json
python ../scripts/export_frontend_data.py   # writes data/metrics.json + data/failures.json
```
